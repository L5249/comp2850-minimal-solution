## Story S1: Prioritisation
**Situation**: When adding a new task to the list 
**Motivation**: Tasks need to have a way to add priority to them
**Outcome**: rist of forgetting key tasks is mitigated
**Underlying need**: unorganised long lists can cause tasks to be forgotten 

**Evidence**: P-A (notes L27,L38)
**Inclusion risk**: ADHD, memory impairments 
**Type**: Job Story(situation-specific)

---

## Story S2: Subtasks and heirarchy
**Situation**: if a task became too large and actually should have many smalled steps 
**Motivation**: Allow for each task to be broken into subtasks
**Outcome**: so progress of a large task can be tracked without smaller tasks beign forgotten
**Underlying need**: users can be overwhelmed by one large task

**Evidence**: P-B (notes L113,L125)
**Inclusion risk**: Cognitive Overload, working-memory limitations
**Type**: Job story (situation-specific)

---

## Story S3: Keyboard only
**Situation**: If mouse stops working or trying to work quicker with keyboard shortcuts
**Motivation**: use every feature with just a keyboard
**Outcome**: continue working at the same speed even if some hardware is broken
**Underlying need**: mouse only actions break if mouse breaks, which means users who only use a keyboard are limited.

**Evidence**: P-B (notes L137)
**Inclusion risk**: Motor impairments, repetitive strain, injuries 
**Type**: Job story (situation-specific)

---

## Story S4: Visibility and contrast
**Situation**: when reading tasks in bright sunlight,can't see them in light mode
**Motivation**: high contrast so in the sunlight you can still read it
**Outcome**: see tasks clearly without straining
**Underlying need**: in sunlight, text thats not standing out is hard to read

**Evidence**: P-A (notes L60), P-B(notes L149))
**Inclusion risk**: low vision, glare sensitivity, vision impairments
**Type**: Job story (situation-specific)

---

## Story S5: Confirmation Feedback
**Situation**: When tasks are added, edited or deleted
**Motivation**: Immidiate confirmation and feedback is needed 
**Outcome**: So the user knows if the action is complete without having to double check or repeat
**Underlying need**: uncertainty causes exessive double checking of lists.

**Evidence**: P-B(notes L113)
**Inclusion risk**: Memory Impairments, Anxiety, ADHD
**Type**: Job story (situation-specific)
